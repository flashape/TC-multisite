Add/Edit a uBillboard

Here you can create new, or edit existing uBillboards. We will go through the options on this page:

uBillboard Options

Title
Allows you to assing a name for your uBillboard, in case you have multiple. It is recommended to use the default name though.
Options - Preview
Allows you to preview the billboard. Will save your changes into a temporary billboard, so you need to make sure you save your changes using the "Save uBillboard" button. This preview also does not necessarily have to look exactly the same as in the frontend. This is because Preview does not load your theme's CSS, and therefore it will not reflect on this previewed uBillboard.
Options - Width
Specifies uBillboard width in pixels. This width represents the width of the actual stage area, where all the slides are. If you have either thumbnails or controls set to "outside", the actual total space taken by uBillboard will be more. If "Apply Automatic Resizing" is enabled, images will be resized to fit this width.
Options - Height
Specifies uBillboard height in pixels, generally the same rules apply as to Width.
Options - Shuffle Slides
Will randomize the order in which slides appear on each page load.
Options - Autoplay
Will automatically start playing once all slides have been loaded.
Options - Square Size
Some of the animations are based around relatively small squares that are animated separately to achieve beautiful transition. You can control the dimension (in pixels) of these squares using this option. Please note: The smaller the dimension, the smaller the squares. The smaller the squares, the more of them are needed to fill the uBillboard space. Too many squares can cause performance issues, general rule of thumb with this option is to keep it on a value that, given your uBillboard Width and Height will generate 50 - 100 squares. [50 > (width / square size) * (height / square size) > 100]
Options - Style
Optional, it is the default style for your content boxes with slide descriptions if you have any. These can be then overriden with the per-description box options in the Content Editor
Options - Delete
Deletes this uBillboard. Not undoable.
Options - Save uBillboard
Saves uBillboard overwriting any uBillboards with the same name.

Slide Order

Slide Order
Each box represents a slide. Drag them around, when you drop, slides on the left will be automatically moved/renamed. You still need to save uBillboard for the changes to take effect on the frontend.

Controls

Show Timer
Will show a countdown circle in the top right corner of the slide that will show the time remaining before the next slide is shown. This will not appear if you don't have Autoplay enabled or don't press Play. This feature is not supported on browsers that don't support Canvas - that is the IE.
Controls Skin
Allows you to choose the skin you want to use with your uBillboard instance. There is a variety of skins with different controls layouts and behaviors as well as look & feel
Controls Position
Controls can be positioned either inside of the uBillboard bounds defined by Width and Height options, or outside. Generally where exactly are they positioned is controlled by the Controls Skin, but you can decide if you want them covering your content. The controls position is useful when your controls overlap with Thumbnails. In that case you can position either the controls, or thumbnails outside or inside so that the two don't overlap.
Show Controls
This option controls the display of Next/Prev buttons. You can either show them at all times, on mouse hover over uBillboard, or never.
Show Play/Pause
Similar to "Show Controls", but controls the Play/Pause button.
Show Paginator
Similar to "Show Controls" and "Show Play/Pause", but controls the paginator. Paginator is most of the time displayed as bullets that each represent the corresponding slide, but in some skins in can take other forms.
Show Thumbnails
Similar to "Show Controls" etc. Thumbnails are small images that represent the content of the slide they point to. When you click a thumbnail, you will be taken to the corresponding slide.
Thumbnails Position
Thumbnails can be positioned on all four edges of uBillboard.
Thumbnails Inside
Similar to Controls Position, thumbnails can be positioned inside or outside.
Thumbnail Width
Width if the thumbnail image in pixels
Thumbnail Height
Height if the thumbnail image in pixels
Thumbnail Hover Color
The border color of each thumbnail

Slide Options

Background Image URL
The main field. You should almost always have an image for your slide. You can select from all images in your Media Library. A preview will be displayed on the left hand side of the slide edit box.
Apply Automatic Resizing
Uses TimThumb to automatically resize your image to the dimensions of uBillboard. This feature can be problematic on some servers, if it does not work for you, see the link above, make sure your cache folder is writable, and if all else fails feel free to contact us.
Use Relative Paths
Will call TimThumb using a relative path to the image specified above instead of an absolute path. On some servers it helps to turn this off. In particular, we've had that experience on a Windows server. But it should only be necessary if TimThumb isn't working for you.
Background Color
Fallback background color for the slide, in case you have transparent images or videos in your slide.
Transparent Background
Related to the above, but since you can not use color picker to select a transparent color, we have this checkbox.
Content Type
Allows you to add additional content overlay on top of your slide. You can select:
  • No Content - just the image
  • Content Editor - Special feature that allows you to add text to your slide, discussed below
  • Embedded Content - allows you to embed YouTube or Vimeo video
  • Blog-based Dynamic Content - allows you to link your slide to the Blog
No Content
Only the image will be displayed.
Content Editor
Content editor allows you to visually position multiple boxes with content ( Textile or plain HTML) on top of your slide.
Text Evaluation
Decides how will the content of the boxes in the Content Editor be treated.
Embedded Content
Specify the URL of the content you want to embed and we'll take care of the rest.
Blog-based Content
Will pull content from the Blog. Content will be displayed on top of the slide, if Featured image is available, it will be used instead of the image specified in "Background Image".
Blog Offset
Blog entries are sorted from the most recent to the least recent. Offset specifies which article to use. Offset of 0 means that the most recent article will be used. Offset of 1 means that the second most recent aricle will be used.
Link URL
The whole uBillboard can be wired to open a specific link. If you specify a link here, it will be used when this slide is currently being shown and the user clicks on an area of uBillboard that does not trigger another functionality (like go to Next page or similar).
Transition Delay:
If Autoplay is active or the user has started the playback manually, this value will determine how much time will pass before the next slide is animated in.
Transition:
Allows you to select an effect that will be applied when switching from the next slide
Transition Direction:
The mode of application of the transition effect. The combination of these two settings determines how will the transition look. There are some combinations that don't make sense and therefore are not implemented. These will fall back to a default transition direction for the selected transition.
