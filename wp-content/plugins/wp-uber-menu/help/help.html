About UberMenu

UberMenu integrates directly with the WordPress 3 Navigation Menu System to give you a seamless user experience. UberMenu adds new options to the normal Menu Items to give you extra flexibility and customization options.

Requirements

There are several requirements for use of UberMenu

  • WordPress 3
  • Use of the WordPress 3 Menu System
    This can be accessed through the Control Panel of any WordPress 3 install via Appearance > Menus
  • A WordPress 3 Menu System-Compatible Theme
    The WordPress theme you use must support WordPress 3 Navigation Menus

How can I tell if my theme is compatible with WordPress 3 Menus?

Your theme must properly implement the WordPress 3 Custom Menu System in order to support WordPress out of the box (note that you can still use UberMenu without a properly supported theme, but you'll need to add some PHP). To check if your theme supports WP3 Menus:

  1. Log into the WordPress Admin area.
  2. Navigate to Appearance > Menus. If this option is not available, WP3 Menus are not supported.
  3. In the Menus panel, look at the Theme Locations meta box. It will tell you if your theme supports WP3 Menus.
Theme Supports WP3 Menus Theme Does NOT Support WP3 Menus
Your theme supports 1 menu. Select which menu you would like to use.
The current theme does not natively support menus, but you can use the "Custom Menu" widget to add any menus you create here to the theme's sidebar.

Known Exceptions:

  • Artisteer does not call wp_nav_menu, so it does not support UberMenu out of the box. You'll need to use UberMenu Easy Integration with Artisteer.
  • Update: As of Headway version 2.0.11, I am informed that Headway now uses the standard WordPress Menu System, so it should now be much easier to integrate! Headway does not support theme locations, so you'll need to follow special instructions to set it up if you want to use multiple menus: How to integrate Headway

Installation & Setup

Install the Plugin

  1. Decompress the .zip file you downloaded from CodeCanyon.
  2. Find the wp-uber-menu folder.
  3. Upload the wp-uber-menu folder to your wp-content/plugins directory.
  4. Navigate to your Control Panel: Plugins
  5. Under UberMenu, click Activate

What did this do?

Installing the Plugin doesn't automatically change any navigation menus - so it won't mess up anything on your site, either. Once you install the plugin, you'll get:

  • A new Control Panel Menu under Appearance > UberMenu
  • New options for Menu Items in the Appearance > Menus Control Panel
  • A new meta options box labeled Activate Uber Menu Locations in Appearance > Menus

Setup the Menu

  1. Navigate to Appearance > Menus
  2. Create a Menu, if you don't already have one
  3. Add Menu Items to your Menu. See UberMenu Structure for information on structuring your menu.
  4. Activate the MegaMenu option on the top level menu items
  5. Save your Menu
  6. Set the Menu Location of your Menu in the Theme Locations Meta Box
  7. Activate the Menu Location as a Mega Menu under Activate Uber Menu Locations
  8. Customize your Mega Menu Items

Remove Residual Styling (optional)

Sometimes, even after activating UberMenu, some of the theme's default menu styles will still affect the menu - often negatively. These styles conflict with the UberMenu styles, and are known as "residual styling". To make UberMenu function like the demo, we need to neutralize this effect. If you experience this issue with your theme, this section will help you resolve the issue. Otherwise, you can skip to the next section.

Understanding the Issue Skip to the solution

The most common scenario with residual styling is that a theme uses a hard-coded wrapper div around the navigation menu with a particular ID. Like this:

<div id="nav">
   <?php wp_nav_menu( array( 'theme_location' => 'main' ) ); ?>
</div>

Which produces the following HTML structure:

<div id="nav">
   <ul id="default-menu" >
      <li><a></a></li>
      <li>
         <a></a>
         <ul>
            <li>...
         </ul>
      </li>
   </ul>
</div>

The default menu styles for the theme would usually look something like this:

#nav ul{
  /* Styles for menu bar */
}
#nav ul li{
  /* Styles for top level items */
}
#nav ul li a{
  /* Styles for top level links */
}
#nav ul li ul li{
  /* Styles for sub menu items */
}

When UberMenu is activated, we end up with this structure:

<div id="nav">
   <div id="megaMenu"> 
      <ul class="megaMenu"> 
	/* UberMenu Structure*/ 
      </ul>
   </div>
</div>

And as a result, some of the style selectors for #nav still apply.

To neutralize these styles we have two choices (pick one):

  1. Remove the original menu styles from the theme
  2. Change the ID of the wrapper (#nav) div

Removing the original menu styles from the theme

Removing the original menu styles from the theme is the best way to go for performance reasons, but it is slightly more involved than just chaning the ID in most cases.

First you'll need to determine the ID of the root item for the menu (your theme's version of #nav). Look at the HTML or use Firebug to inspect the menu and determine the ID of the div that wraps #megaMenu.

Next you'll need to track down where the menu styles are located. Most often this is in the theme's style.css file. Inspecting the menu Firebug makes tracking down the CSS very easy.

Once you find the styles for the default menu, remove them (or comment them out). Often, the styling for the wrapper div itself (i.e. #nav{ /* this code */ } ) handles the positioning of the menu, so you may want to leave it intact.

Changing the ID of the wrapper div

The simplest "silver bullet" solution is to just change the ID of the wrapper div - that is, the div that all the menu item selectors are based on. In the example, it is #nav

A really easy test to determine whether this will work is to use Firebug to inspect the DOM and find the element that wraps the #megaMenu div. (This is most often the div used to target the menu, but it may be a more outer wrapper). Simply highlight the wrapper div and change the ID to something else in the Firebug inspector. Test the menu by hovering again. If the residual styles are gone, that's the ID you're looking for.

Once you've identified the div whose ID you need to change, you need to find it. Most often it is in your theme's header.php file. If not, you can search through your theme's files for the string 'wp_nav_menu' or the id="{your id}". When you find the line, just edit the ID and change it to any other string, for example, change:

<div id="nav">
   <?php wp_nav_menu( array( 'theme_location' => 'main' ) ); ?>
</div>

to

<div id="nav_uber">
   <?php wp_nav_menu( array( 'theme_location' => 'main' ) ); ?>
</div>

One final step. You may find that doing this removes the residual styles, but also changes the placement of the menu. The solution for this is to find the CSS for the wrapper div in the theme's CSS and change the selector from #nav to your new selector (e.g. #nav_uber). This is generally the only style you'd need to change, as it most often contains the positioning code.

Setup Widgets

  1. UberMenu provides an unlimited number of Widget Areas. Set how many you want to have via Appearance > UberMenu > Advanced > Number of Sidebars. You can always increase this later.
  2. Navigate to Appearance > Widgets and add widgets to the UberMenu Widget Areas
  3. Navigate to Appearance > Menus and expand a Menu Item (create it first if you haven't already), preferably a second-level item (though third-level+ will also work)
  4. Click Show/Hide UberMenu Options
  5. In the Display a Widget Area select box, select the Widget Area to display in this Menu Item
  6. Save the Menu
  7. Now UberMenu will load the Widget Area into your Menu. You can update the Widgets in the future without needing to change the Menu again.

Installing Updates

Installing updates is just like installing UberMenu for the first time. It's always a good idea to back everything up before you upgrade, especially if you have made any changes to UberMenu's files, as the upgrade will overwrite all files. UberMenu settings set through the control panel will all remain intact.

Backing Up

  1. It is always a good idea to back up your plugin before making any changes
  2. Open your FTP program and FTP into your site and navigate to the wp-content/plugins directory.
  3. Download the wp-uber-menu folder and save it as a backup
  4. If you have edited any files, especially the custom.css file, backing this file up is crucial

Upgrading

  1. Download the new version of the plugin from CodeCanyon
  2. In your browser, log into the WordPress control panel.
  3. Navigate to Plugins
  4. Deactivate UberMenu
  5. Switch to your FTP program, FTP into your site and navigate to the wp-content/plugins directory.
  6. Delete UberMenu (the entire wp-uber-menu folder).
  7. On your desktop, decompress the zip file you downloaded from CodeCanyon
  8. Open the unzipped folder, which will contain a new wp-uber-menu directory
  9. FTP the new wp-uber-menu folder into the plugins directory.
  10. If you saved an edited copy of your custom.css file as a custom preset, upload it to wp-uber-menu/styles/skins directory (overwrite the new custom.css file).
  11. Switch back to your browser and go back to the Plugins panel
  12. Reactivate UberMenu by clicking Activate

Note: If you are running a local server, just copy the files rather than FTPing

New WordPress Menu Options

Activate UberMenu Locations

You'll notice a new Meta Options box in the Menus Control Panel. This panel determines whether a particular menu location will be megafied. The location of your menu must be activated in order to megafy the menu. This allows you to have certain one location megafied while others remain normal menus.

Please note that you can only have one Mega Menu per page. You could have 2 mega menus, but only if they never occur on the same page.

The UberMenu Options Panel

The UberMenu Options Panel is the place to customize your UberMenu. It is accessible via your Control Panel, under Appearance > Uber Menu. With it you can:

  • Configure settings like orientation, transition, trigger event, description display, image size, and more
  • Set how you'd like to style your Menu
  • Select Style Presets
  • Create Custom Styles and generate CSS
  • Preview your Menu

Top Level Menu Items

For a horizontally-aligned menu, the top level menu items are listed in the row of main links in the navigation menu. For a vertically-aligned menu, these items are aligned in a column. The top level menu items are always visible.

Example of Top Level Menu Items: Electronics | Books | Sports & Outdoors | Clothing

Second Level Menu Items

The second level menu items (those that are indented once below top level menu items) are treated as headers within the sub menus. They contain third level menu items to for sub menu groups. These groups are aligned as columns within the sub menu, with the second level menu items as the headers at the top of each column.

Example of Second Level Menu Items, sub-menu of Electronics: Computers | TVs | Cell Phones | MP3 Players

Third Level Menu Items and Below

All menu items deeper than level 3 are treated as level 3 menu items. These items are the content links within their parent categories. Sibling items are aligned vertically in list-fashion below their column headers (second level menu items).

Example of Third Level Menu Items, under Electronics > Computers: Laptops | Desktops | Netbooks | Accessories

Adding Content to Your UberMenu

To create an UberMenu, follow the Setup Instructions, and then add content to your Menu. Items are added via the normal WordPress 3 Menu Management mechanism. For more information, read this.

Normal Flyout Menus

Even in an UberMenu, you can still create a normal flyout menu. Just make sure that Activate Mega Menu is NOT checked on the top level menu item. Add items as usual. Note that if the top level item is not a Mega Menu, the Mega Menu options are non-applicable.

Mega Menus - Normal Link Content

Adding normal link content to a Mega Menu is as simple as creating a flyout menu. Just make sure that Activate Mega Menu IS checked on the top level menu item. Second level items create column headers and third level items fill the content below those headers. See UberMenu Structure for more information on how the structure works. See Menu Item Options for more information on the available Mega Menu options.

Mega Menus - Link Content with Images

UberMenu makes it easy to apply an image to each menu item, in any level. The image is placed to the left of the menu item text. Images are automatically resized. You can control how this is done (via TimThumb or normal browser scaling) and to what size the images are scaled through the UberMenu Options Panel.

Here's how to apply an image to a Menu Item:

  1. Add a Menu Item
  2. Click Show/Hide UberMenu Options to display the advanced options.
  3. Click Set Thumbnail
  4. A popup dialogue appears. If the Select Files button is not available, click the From Computer tab
  5. Click Select Files
  6. Upload or select the image
  7. Click Use as Featured Image - *this is critical
  8. Change any other settings, like title, etc, and click Save All Changes
  9. Close the popup
  10. You should now see a preview of your image in the Menu Item
  11. Save your Menu

Mega Menus - Link Content with Images - No Text

If you want to create image-only links (icons but no text), you'll need to use a special tag. In the Navigation Label box within a Menu Item, enter the following:

--notext--

Note that if you try to save the item with a blank Navigation Label, the item will be removed

Mega Menus - Link Content with Descriptions

Adding descriptions to your Menus is simple with UberMenu. Here's how to set them up:

  1. Activate the description levels you'd like to use in the Appearance > UberMenu control panel. Under Descriptions, turn on the levels you'd like to activate. Note: this is done because descriptions are often set by default to the excerpt or full post text - if you only want to display descriptions on a particular level, this saves you from deleting all of the menu item descriptions on other levels.
  2. In the Appearance > Menus panel, click the Screen Options tab in the upper right corner of the screen.
  3. In the Show advanced menu properties section, check the Description box.
  4. Now in every Menu Item box, there should be a text area labeled Description. Enter text to set a description.
  5. Save your menu.

Content Beyond Links: Contact Forms, Blog Posts, Maps, and More!

UberMenu can take just about any type of content, via 2 techniques: Widgets and Shortcodes (or a combination of the two).

Adding Content Via Widgets

Adding content via widgets is easy, but takes several steps. The basic process is to (1) Create a Menu Item (2) Set the Menu Item's Widget Area (3) Populate the Widget Area with Widgets. You should think of a single menu item, which contains a single Widget Area, as a row in the second level of the menu. Multiple widgets in a single widget area create columns within that row.

  1. Set the number of Widget Areas you are going to use via the Control Panel Appearance > Uber Menu > Advanced > Number of Sidebars. It must be set to at least 1 to use widgets.
  2. Save the UberMenu options with your new Number of Sidebars.
  3. Create a Menu Item (or edit an existing menu item) - second level menu items work the best
  4. Expand the UberMenu Options and select a Widget Area in the dropdown next to Display a Widget Area
  5. Save your Menu
  6. Navigate to Appearance > Widgets
  7. Add Widgets to the Area that you selected in step 3, UberMenu Widget Area X
  8. Note that you may use shortcodes within text widgets.
  9. Save your widgets

Adding Top-Level Widgets (new in UberMenu 1.1)

Note that by default, top-level widgets are deactivated. To activate top-level widgets:

  1. Navigate to Appearance > UberMenu
  2. Expand Advanced and activate Allow Top-Level Widgets
  3. Save Options

Once you have top-level widgets activated, you'll be able to use widgets in top level menu items as described above. Here are some formatting tips:

  1. Setting the Menu Item's Navigation Label to --notext-- will remove the Menu Item text
  2. Placing a single space in the Widget's Title will remove that formatting (this works well with the Search box)
  3. The default WordPress searchbox widget has been styled via the presets, but you may need to apply custom styles to other widgets to format them properly for the menu

Adding Content Via Shortcodes

Adding content via shortcodes is easy, but formatting is sometimes unpredictable. You may need to apply custom CSS styles to your menu in order to make your shortcodes display the way you'd like them to - especially custom widths.

  1. In the Control Panel under Appearance > Uber Menu > Advanced, turn Allow Content Overrides on
  2. In Appearance > Menus, add an item to your menu
  3. Expand the UberMenu Options for the Item.
  4. Enter your shortcode in the Content Override box.
  5. Save your menu.
  6. Apply custom styles if necessary

Included Shortcodes

UberMenu comes with some useful shortcodes.

Columns shortcode

Basic link content gets columnized automatically, as does widgetized content. While this automatic column structure takes care of most use cases, sometimes you want more advanced column structures with uneven columns. The columns shortcode gives you this ability. Creating Your Own Styles

Of course, you're not limited to only the preset styles that come with UberMenu. You can get as creative as you want. UberMenu styles can be controlled via CSS. Through the UberMenu Control Panel

  2. The UberMenu Style Generator will be displayed.
  3. You can set colors, font sizes, and border radii, etc.

UberMenu Easy Integration - Using UberMenu with Themes that do not support WordPress 3 Custom Menus

UberMenu works by applying filters to a WordPress 3 Custom Nav Menu. These menus are inserted into the page via your WordPress Theme. Normally, UberMenu automatically replaces your theme's WP3 custom menu, when WP3 custom menus are supported. If your theme does not already support custom menus, you can add support by turning on UberMenu Easy Integration and adding a line of PHP code. Note: you still need to be running WordPress 3.0 or above for this to work.

1. Activating WordPress 3 Custom Menu Support

UberMenu 1.1+

  1. Navigate to Appearance > UberMenu
  2. Under Theme Integration, enable Easy Integration
  3. Click Save Options

2. Placing the Menu

Now that the theme supports WP3 Menus, we'll need to actually insert the menu somewhere. Most likely, you'll want to replace your theme's old menu, which is usually a call to wp_list_pages or wp_page_menu, though it can vary depending on your theme. Once you've removed the old menu and any surrounding <ul> tags, you'll want to insert the following PHP code where the menu should be displayed. Usually this would be inserted in you theme's header.php file, but again this may vary depending on your theme.

Code for UberMenu 1.1+

<?php uberMenu_easyIntegrate(); ?>

Note: The above menu call should not be located inside <ul> tags, even if your theme's default menu is

Completing the above two steps should successfully enable your theme to work with UberMenu, giving you an "UberMenu" theme location in your menus panel. Now just configure UberMenu as usual!

My Mega Menu takes the full width of the page. I want it to be centered to the width of my content. How can I do this?

Note that as of UberMenu 1.1, you can change the width of the menu via Appearace > UberMenu > Advanced > Menu Width

This means that the container div that your menu is located in is full width. UberMenu conforms to the width of its container so that its settings don't override your theme styles. If you want to center your menu to your content, you'll need to apply some simple CSS:

#megaMenu{
	width: 960px; /* Your page width */
	margin: 0 auto;
}

Where 960px is replaced with the width of your site content. You can add this style to any CSS file. If you're using a preset, you might want to copy the preset code to the Custom Preset file, then add this style to the top. That way, if you upgrade the plugin in the future, you won't overwrite your changes.

Note that you could alternatively apply this style to the container of the menu (the div that surrounds it). For example, if the div's ID is access, you could apply this style to a style sheet in your theme:

#access{ /* This is the container div that wraps the #megaMenu div */
	width: 960px; /* Your page width */
	margin: 0 auto;
}

My sub menus are being hidden behind other content. How can I fix this?

This is usually a z-index issue with your theme's container divs. There is a simple fix, but it must be applied to the appropriate containers. Specifically, the adjacent containers of the overlapping elements. For example, in this situation:

<div id="wrap">
  <div id="nav-container">
    <div id="megaMenu">
      ...
    </div>
  </div>
  <div id="content">
    <div id="overlapping-element"></div>
  </div>
</div>

You would need to apply these CSS styles:

#nav-container{
  position:relative;
  z-index:20;
}
#content{
  position:relative;
  z-index:10;
}

Put another way, the styles need to be applied to the relevant children of the first common ancestor of the target (overlapping) elements. Above, the first common ancestor is #wrap, and its children are #nav-container and #content. Sometimes, the adjacent containers can be several levels up the DOM tree, so it is not necessarily the parent items that need to be positioned.

How can I change the animation speed of my submenus?

By default, the slide and fade animations are executed at the "normal" speed. However, you can make the speed "fast", "slow", or a particular duration in milliseconds by changing the javascript.

  1. In a text editor, open wp-uber-menu/js/wpmegamenu.dev.js
  2. You'll see this code around line 143:
    	if($u.wpmega.trans == 'fade') $subMenu.fadeIn();
	else $subMenu.slideDown();
  3. Pass the fadeIn() or slideDown() functions a duration parameter to change the animation speed, for example:
    	else $subMenu.slideDown('fast');
    You could also use 'slow' or an integer like 2000 (2000 = 2 seconds).
  4. Save your file
  5. You now have two options:
    1. Switch your UberMenu into debug mode under Appearance > UberMenu > Advanced > Debug Mode. This will just make UberMenu use the development javascript file you just edited instead of the minified file. (Note this is a good way to test your changes).
    2. Or you can copy the contents of the file you just edited into wpmegamenu.min.js (replace the existing contents) and save that instead
  6. Optionally, you can minify your new code by using Google's Closure Compiler and put that code in the wpmegamenu.min.js file.

I receive a Parse Error when activating the plugin. Why?

If you receive the following error when activating UberMenu:

Plugin could not be activated because it triggered a fatal error.
Parse error: parse error, unexpected T_STRING, expecting T_OLD_FUNCTION or T_FUNCTION
or T_VAR or '}' in /var/www/html/wp-content/plugins/wp-uber-menu/wpMegaWalker.php 
on line 12

it is because your server is running PHP4, but UberMenu requires PHP5.2+ (see requirements in product description). Not to worry, there are two simple solutions to make it work:

  1. Upgrade your server to PHP5.2.x. This is the recommended solution, as PHP5 has advantages over PHP4. Usually this is either an option in your server administration control panel, or your hosting provider can do this for you.
  2. If you must run PHP4, you can just delete line 12 from plugins/wp-uber-menu/wpMegaWalker.php, which removes the PHP5 code. Everything should still run smoothly.

When I use the columns shortcode, I get a PHP Warning. What gives?

The columns shortcode uses regular expressions to intelligently determine your columns without the need to identify each column's index. This requires the use of the PHP pcre package, version 7.8 or later. The following error:

PHP Warning: preg_match_all(): Compilation failed: unrecognized character after (?<

indicates that you are running an older version of pcre. To solve this error, upgrade your server's pcre package. You can contact your hosting provider for specific details.

Where should I put my custom CSS?

You can put your CSS styles in any stylesheet that makes you happy. Your choice depends on an inherent trade-off in ease vs. maintainability.

Option 1: UberMenu's custom.css

This is the recommended solution. Remember, custom.css is only loaded if you have the "Custom" preset selected in the UberMenu options. You'll need to copy your preferred preset styles into this file, and add your custom styles as well. See Creating your own styles via CSS for details. This is the best place from a maintenance perspective, because when you upgrade you'll just need to preserve the custom.css file.

Option 2: A special place where you keep custom CSS.

If you have created (or if your theme provides) a special location for custom CSS styles, this is a great place for custom UberMenu styles as well.

Note: Just keep in mind that this stylesheet is likely loaded before UberMenu's, so if you're trying to override existing styles you'll need to either use more specific selectors or use the !important flag in your styles.

Option 3: Your theme's style.css

You might already be putting your customizations in your theme's style.css. This is fine, just make sure you keep track of them. It's best to keep them all in one place so you can easily transfer them if you upgrade your theme. Also, see the above note regarding specificity.

Option 4: UberMenu's preset files

This is discouraged, because it will make it difficult to upgrade. If you want to edit or add to one of these files, copy it to custom.css first as described in Option 1.

Option 5: UberMenu's basic.css

This is a core file. It should never be edited unless you have no intention of ever upgrading UberMenu

Pick your poison! :)

Karma

UberMenu integrates well with Karma except when it comes to templates with additional menus. If you use a template like Left Nav + Sidebar, you'll find that both the header nav menu and the sidebar nav menu become UberMenus.

The issue arises because Karma defines both menus as the same menu theme location. That is, both the menu in the header and the menu in the sidebar are defined as "Primary Navigation" by Karma. As a result, there is no distinction to be made between the two menus (they are really the same menu, printed twice), so you end up with two UberMenus.

/* Karma/header.php line 53 */
wp_nav_menu( array(
 'container' =>false,
 'theme_location' => 'Primary Navigation',
 'sort_column' => 'menu_order',
 'menu_class' => '',
 'echo' => true,
 'before' => '',
 'after' => '',
 'link_before' => '',
 'link_after' => '',
 'depth' => 0,
 'walker' => new description_walker())
 );
}

/****************************************************/

/* Sidebar - Karma/functions/global/subnav-left.php line 2*/
$menu_code = wp_nav_menu( array(
 'container' =>false,
 'theme_location' => 'Primary Navigation',
 'sort_column' => 'menu_order',
 'menu_class' => '',
 'echo' => false,
 'before' => '',
 'after' => '',
 'link_before' => '',
 'link_after' => '',
 'depth' => 0,
 'walker' => new sub_nav_walker())
 );

The solution is to define a menu theme location for each real menu location within the templates (header, nav, etc). Really, we just want to make sure that the header menu is an UberMenu and nothing else. So the simplest solution is to use UberMenu Easy Integration for the header menu. This will automatically define a new theme location called "UberMenu". Switch it on in Appearance > UberMenu > Theme Integration > Easy Integration, then add the appropriate code:

/* Replace the entire wp_nav_menu() call in header.php line 53 with uberMenu_easyIntegrate() */

/* Comment this out: */
/*wp_nav_menu( array(
 'container' =>false,
 'theme_location' => 'Primary Navigation',
 'sort_column' => 'menu_order',
 'menu_class' => '',
 'echo' => true,
 'before' => '',
 'after' => '',
 'link_before' => '',
 'link_after' => '',
 'depth' => 0,
 'walker' => new description_walker())
 );*/

/* Add this: */
uberMenu_easyIntegrate();

Once you've done that, just go to Appearance > Menus and set the appropriate menu for the UberMenu location, and activate that location (deactivate the others). You can set whatever menu you want for the Primary Navigation location, which will now be used for things like sidebars (you can use the same menu here, it'll just be printed normally rather than as an UberMenu.

So now,

  • UberMenu (location) = header menu
  • Primary Navigation = other menus
